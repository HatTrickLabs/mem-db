MemDb Docs....
------------------------
- Who (are we) => Hat Trick Labs
  - Simple yet powerful and efficient software tools.
  
- What (is MemDb) =>
  - A lightweight database / data cache engine.
  - Self contained, zero external dependencies outside core .net
  - Runs embeded in process (no dedicated engine process).
  - Entire data footprint in RAM.
  - File backed (data persisted to disk)
  - Single compact library (~104KB HatTrick.MemDb.dll)

- Why (did we build it) =>
  - Need for a super simple yet powerful embedded data storage.
  - File backed data structure (persisted to disk).
  - Entire data set stored in RAM for fast reads (can be used as a file backed cache).
  - Powerful query capabilities using native code (no sql language parsing).
  - Always safe concurrency.
  - Cached data integrity (never insert ref pointers and never return ref pointers).
  - Baked in record level data encryption (AES 256 bit)
  - Online/Hot snapshots.
  - Archive and restore (ability to restore a database back to a histrical point in time).
  - Super simple implementation given default settings.
  - Powerful extensability points when performance and efficiency are a necessity.

Core Concepts
-------------
- Modes (ReadWrite, ReadOnly, AppendOnly)
- Persistence (optional)
  - Files (.db | .map | .lock)
- Flush (timed and manual)
- Cache<T>
  - where T class (ref type)
  - Log structured cache, new versions of records always appended to the end of the cache and old versions marked stale (see Purge)
- Data Integrity via deep cloning (see Serialization)
 - no ref pointers cross bounderies
- Log structured files, new versions of records always appended to the end of the data files and old versions marked stale (see Defragment)
- Default Serialization
  - System.Text.Json.JsonSerializer

Indexes
---------------------
- Configure identity (id) index, this is the natural index: .IndexOnIdentity(bool shouldIndex) 
- Query via identity index (Id) (Hash lookup) O(1)
 - Find(1)
 - FindAll([1,2,3,4,5])
- Configure additional indexes (applied): .ApplyIndex<T>(string indexName, Func<U, T> keyResolver)
  QueryViaIndex<string>(string indexName)
  - IsEqualTo => Hash lookup O(1)
  - IsNotEqualTo => O(log n) * 2
  - IsGreaterThan => O(log n)
  - IsLessThan => O(log n)
  - IsGreaterThanEqualTo => O(log n)
  - IsLessThanEqualTo => O(log n)
  
  Online Cache Maintenance
  ------------------------
  - Log strutured cache does not automatically cleanup stale or deleted data.
    - Use MemDb.Purge(); to remove stale and deleted records from the in memory cache.


Snapshots
----------
- Configure: .SnapshotTo(string path)
- Point in time backups.
- Online funciton, can be executed without closing the database.
- Built from the file system not the Cache (avoid locking on the cache, db still completely operational).
- 

Defragmentation
- Offline process of removing the stale and deleted records from the database files.
- Archive option (see point in time restore)

Point In Time Restore
----------------------
- Restore archived database back to a specific point in time.
